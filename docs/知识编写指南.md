# 📝 知识编写指南

本文档说明如何编写和修改「人生必修课」项目中的知识条目。

## 📁 文件结构

每个知识条目是一个独立的 JSON 文件，存放在 `src/data/knowledge/` 目录下。

**命名规则：** 文件以知识的中文名命名，例如 `有效阅读.json`、`睡眠健康.json`。

## 📋 JSON 结构

```json
{
  "id": "中文ID",
  "title": "显示标题",
  "summary": "一句话概述",
  "branchIds": ["所属分支"],
  "lesson": "不学会损失什么（直击痛点的一句话）",
  "journey": ["心路历程1", "心路历程2", "心路历程3"],
  "knowledge": {
    "core": "核心知识",
    "detail": "详细说明"
  },
  "selfTest": ["检验标准1", "检验标准2", "检验标准3"],
  "related": ["关联知识ID"],
  "children": ["子条目ID"],
  "resources": [
    { "type": "book", "title": "书名", "author": "作者", "url": "..." }
  ]
}
```

## 🧩 字段详解

### id（必需）
知识条目的唯一标识，**必须与文件名一致**（不含 `.json` 后缀）。

### title（必需）
显示在页面上的标题，可以包含 emoji 增强可读性，例如 `📚 基础能力`。

### summary（必需）
一句话概述本条目的核心内容，让读者快速理解这是关于什么的。

### branchIds（必需）
所属分类分支 ID 数组，来自 `src/data/branches.json`。一个知识可以属于多个分支。

### lesson
**教训**板块，一句话直击痛点：不学这个知识会损失什么。

**写作要点：** 让读者产生危机感。不要笼统说"很重要"，要具体——不学健身→腰酸背痛体态差；不学理财→工资永远不够花。

### journey（心路历程）
第一人称的认知转变过程，通常 3 条：
1. 曾经不屑一顾/做错的事
2. 转折点（遇到了什么、学到了什么）
3. 现在的认识和改变

**写作要点：** 真诚的"悔悟感"，让读者产生"这不就是我吗"的共鸣。

### knowledge（内容）
核心知识内容，包含两个子字段：

| 子字段 | 说明 | 写作要点 |
|--------|------|----------|
| `core` | 核心知识 | 提炼最精华的内容，让读者读完后立刻能上手。可以用列表、金句、关键原则等形式 |
| `detail` | 详细说明 | 深入的展开说明，可以是方法论细节、扩展阅读、注意事项等。可选字段 |

### selfTest（检验标准）
3-5 个自测问题，帮助读者判断自己是否掌握了该知识。

**写作要点：** 问具体的问题，而非笼统的"你学会了吗"。好的自测问题应该是"如果你能做到 XX，说明你掌握了"。

✅ 好的例子：
- 你能放松地站 10 分钟而不觉得膝盖疼吗？
- 你知道自己每天的时间主要花在哪里吗？

❌ 差的例子：
- 你掌握了吗？

### related（相关知识）
弱关联的其他知识条目 ID 数组。用于建立知识网络，在详情页侧边栏显示为"相关知识"。

**规则：** 关联应该是双向有意义的——如果 A 关联了 B，B 最好也关联 A。

### children（子条目）
父子层级关系，数组中的 ID 是当前条目的直接子级。用于构建知识树的上下级导航。

### resources（资源索引）
学习资源列表，每条资源包含：

| 字段 | 必需 | 说明 |
|------|------|------|
| `type` | 是 | 资源类型：`book` / `video` / `course` / `article` / `website` |
| `title` | 是 | 资源名称 |
| `author` | 否 | 仅图书资源需要 |
| `platform` | 否 | 平台名称，如"B站"、"Coursera" |
| `url` | 否 | 链接地址，暂无链接用 `"..."` 占位 |

## 🔗 知识关联方式

项目支持三种关联关系：

### 1. 顺序关系（前后置知识）
定义在 `src/data/relations/*.json` 中，通过 `from → to` 有向链接表示学习顺序。
- `default.json`：通用学习路径
- `programmer.json`：面向程序员的精简路径
- 可在 `index.js` 中通过 `setActiveRelationId()` 切换

### 2. 父子关系（上下级）
通过 JSON 中的 `children` 数组定义。父级知识包含子级 ID。
- `getParent(id)`：查找父级
- `getChildren(id)`：查找子级

### 3. 弱关联（相关知识）
通过 JSON 中的 `related` 数组定义。系统会自动计算"被引用"关系。

## 🔧 工具脚本

### 重命名知识条目

```bash
# 单个重命名（自动更新所有引用）
node scripts/rename-knowledge.js <旧ID> <新ID>

# 示例：将"跑步"重命名为"跑步入门"
node scripts/rename-knowledge.js 跑步 跑步入门
```

脚本会自动：
- 重命名文件
- 更新 JSON 内的 `id` 字段
- 更新所有知识文件中的 `related`/`children` 引用
- 更新 `relations/*.json` 中的 `from`/`to` 引用

## 💡 写作原则

1. **不创造知识，只做索引和整合**——引用已有书籍、课程、文章作为资源
2. **真诚胜过权威**——心路历程要真实，教训要扎心，读者才会信服
3. **具体胜过抽象**——不要讲大道理，要讲怎么做、什么感觉、什么后果
4. **一篇文章讲透一件事**——每个知识条目聚焦一个主题，不要贪多
5. **检验标准可执行**——读完能立刻判断自己会不会
